# STM32 CMake Base

Этот репозиторий представляет собой стартовый шаблон для bare-metal разработки под `STM32` на `C/C++` с использованием `CMake`, `arm-none-eabi-gcc`, `CMSIS` и набора собственных драйверов автора. Проект рассчитан на работу из `VS Code`, но не привязан к IDE и может собираться из командной строки.

Основная идея репозитория:

- быстро поднять новый проект под конкретный STM32 без CubeMX;
- генерировать линкерный скрипт под выбранный чип автоматически;
- подтягивать CMSIS/device/startup-файлы напрямую из внешнего репозитория.

Это именно база для старта прикладного приложения.

## Что внутри

На верхнем уровне находятся:

- `base-setup.cmake` — пользовательская конфигурация проекта: имя, чип, список исходников, include-пути, дополнительные define'ы и опции.
- `CMakeLists.txt` — основной сценарий сборки.
- `CMakePresets.json` — готовые пресеты `debug`, `release`, `relWithDebInfo`, `minSizeRel`.
- `cmake/` — тулчейн, вспомогательные функции и логика загрузки CMSIS-файлов.
- `src/` — пользовательский код приложения.
- `inc/` — пользовательские заголовки.
- `no_system_files/` — `syscalls.c` и `sysmem.c` для newlib/newlib-nano.
- `cmsis-core/` — локальная база CMSIS Core headers.
- `cmsis-core/download_files/` — скачанные startup/device/linker/SVD-файлы под выбранный MCU.
- `.vscode/` — готовые настройки задач, прошивки и отладки через ST-Link.

## Что делает проект при конфигурации

Во время `cmake configure` проект:

1. Берёт имя целевого микроконтроллера из `base-setup.cmake` (`DEVICE`).
2. По этому имени определяет семейство, startup-файл, заголовки устройства и объёмы памяти.
3. Скачивает недостающие файлы из репозитория `STM32-base_files`:
   - `startup_common.c`
   - `vector_<device>.c`
   - device headers
   - `system_<series>.c/.h`
   - `SVD`
   - шаблон linker script
4. Генерирует итоговый `.ld` файл под выбранный чип.
5. Генерирует `flash_config.h` для Flash-драйвера.
6. Подключает локальные драйверы из `Drivers/`.

Если чип изменился, логика в `cmake/cmsis-download.cmake` пытается очистить скачанные файлы и часть артефактов сборки, чтобы не смешивать конфигурации разных MCU.

## Для какого сценария подходит репозиторий

Репозиторий удобен, если вам нужен:

- bare-metal проект без HAL;
- контроль над startup/linker/CMSIS на уровне файлов;
- сборка через `CMake` и `Ninja`;
- работа в `VS Code` с `CMake Tools` и `cortex-debug`;
- база для собственной библиотеки драйверов.

## Пример приложения

Текущий `src/main.cpp` — минимальный пример прямой работы с регистрами:

- включает тактирование `GPIOB`;
- переводит `PB7` в output;
- бесконечно переключает `PB7` с программной задержкой.

То есть после успешной сборки и прошивки прошивка просто мигает выводом `PB7`.

В коде также оставлены закомментированные примеры использования драйверов:

- `System::Init()`
- `ClockSystem::Init_calc_pll(...)`
- `System::Enable_CYCCNT()`
- `PIN::SetUp(...)`
- `System::SWOTrace(...)`

## Требования

Для работы с проектом нужны:

- `CMake` версии `3.22+`
- `Ninja`
- тулчейн `arm-none-eabi-gcc` в `PATH`
- `arm-none-eabi-g++`
- `arm-none-eabi-objcopy`
- `arm-none-eabi-size`
- `git`

Для прошивки и отладки дополнительно:

- `STM32CubeProgrammer` CLI (`STM32_Programmer_CLI.exe`)
- `ST-Link`
- при отладке из VS Code: расширение `cortex-debug`
- при работе с CMake в VS Code: расширение `ms-vscode.cmake-tools`

## Быстрый старт

### 1. Выберите целевой микроконтроллер

Откройте `base-setup.cmake` и задайте нужный чип:

```cmake
set(DEVICE "STM32F446re")
```

Примеры:

- `STM32F446re`
- `STM32F429zi`
- `STM32F407vg`

Строка должна совпадать с шаблоном, который понимает логика `cmake/cmsis-download.cmake`.

### 2. Настройте проект

При необходимости отредактируйте в `base-setup.cmake`:

- `prj_name` — имя проекта;
- `USE_DRIVERS` — подключать ли пользовательские драйверы;
- `sources_SRCS` — список ваших `.c/.cpp`;
- `include_DIRS` — include-пути;
- `symbols_c_SYMB`, `symbols_cxx_SYMB`, `symbols_asm_SYMB` — дополнительные макросы;
- `compiler_OPTS`, `linker_OPTS`, `link_LIBS` — пользовательские флаги и библиотеки.

### 3. Сконфигурируйте и соберите

Из корня репозитория:

```powershell
cmake --preset debug
cmake --build --preset debug
```

Для релизной сборки:

```powershell
cmake --preset release
cmake --build --preset release
```

Артефакты попадают в:

```text
build/<preset>/
```

После сборки автоматически создаются:

- `.elf`
- `.hex`
- `.bin`
- `.map`
- `.dis`

## Работа из VS Code

В репозитории уже подготовлены:

- `CMakePresets.json`
- `.vscode/tasks.json`
- `.vscode/launch.json`
- `.vscode/settings.json`

Что уже настроено:

- сборка через активный `CMake` preset;
- прошивка через `STM32_Programmer_CLI.exe` по `SWD`;
- полное стирание чипа;
- запуск отладки через `cortex-debug` и `ST-Link`;
- использование локально скачанного `SVD` файла.

Типичный сценарий в VS Code:

1. Открыть папку проекта.
2. Выбрать configure preset, например `debug`.
3. Выполнить `CMake: Configure`.
4. Выполнить `CMake: Build`.
5. Запустить задачу `STM32Prog: Flash project (SWD)` или `Build + Flash`.
6. Для отладки выбрать одну из конфигураций `ST-Link Launch` / `ST-Link Attach`.

## Прошивка из командной строки

Если `STM32_Programmer_CLI.exe` доступен в `PATH`, можно прошивать так:

```powershell
STM32_Programmer_CLI.exe --connect port=swd --download build/debug/<project>.elf -hardRst --start
```

Посмотреть доступные интерфейсы:

```powershell
STM32_Programmer_CLI.exe --list
```

Полное стирание:

```powershell
STM32_Programmer_CLI.exe --connect port=swd --erase all
```

## Как добавить свой код

Обычно достаточно:

1. Добавить `.cpp/.c` файлы в `src/`.
2. Добавить заголовки в `inc/`.
3. Подключить новые исходники в `sources_SRCS` внутри `base-setup.cmake`.
4. При необходимости добавить include-пути в `include_DIRS`.
5. Если используются драйверы, раскомментировать/добавить нужные модули в список сборки.

## Как переключить проект на другой STM32

1. Измените `DEVICE` в `base-setup.cmake`.
2. Переконфигурируйте проект: `cmake --preset debug`.
3. При необходимости удалите старый каталог `build/`, если в нём остались артефакты от другого чипа.
4. Убедитесь, что `launch.json` и настройки отладки соответствуют новому устройству.

Особенно важно проверить `.vscode/launch.json`: там сейчас в нескольких профилях жёстко прописан `STM32F446RE`.

## Важные особенности и ограничения

- Проект зависит от загрузки файлов из GitHub во время конфигурации. Без доступа к сети первая настройка не пройдёт, если нужных файлов ещё нет локально.
- В `CMakeLists.txt` linker script берётся из `cmsis-core/download_files/linker/`, поэтому итоговая рабочая конфигурация должна находиться именно там.
- Шаблон ориентирован на собственные драйверы автора, а не на HAL/LL.
- В `main.cpp` сейчас используется прямой доступ к регистрам, поэтому начальная частота/инициализация зависят от reset state MCU.

## Что стоит сделать сразу после клонирования

- выбрать свой `DEVICE`;
- проверить, что `arm-none-eabi-*` доступны из консоли;
- выполнить `cmake --preset debug`;
- убедиться, что нужные startup/device/linker-файлы скачались;
- поправить `.vscode/launch.json` под свой MCU и способ отладки;
- заменить демонстрационный `src/main.cpp` на свою прикладную логику.

## Куда смотреть в первую очередь

Если вы хотите быстро понять проект, начните с этих файлов:

- `base-setup.cmake` — вся пользовательская конфигурация.
- `CMakeLists.txt` — как именно собирается target.
- `cmake/cmsis-download.cmake` — откуда берутся startup/header/linker-файлы.
- `src/main.cpp` — минимальный пример.
- `Drivers/src/*.hpp` — доступные драйверы и их API.

## Итог

Этот репозиторий — удобная база для собственных bare-metal проектов под `STM32`, если вы хотите:

- собирать всё через `CMake`;
- не зависеть от CubeMX;
- использовать CMSIS и собственные C++-драйверы;
- иметь готовый старт для сборки, прошивки и отладки в `VS Code`.

Для первого запуска главное — привести `DEVICE`, startup/linker/SVD и debug-конфигурацию к одному и тому же микроконтроллеру.
